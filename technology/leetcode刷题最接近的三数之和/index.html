LeetCode刷题——最接近的三数之和

 2021.12.2  2021.12.7  1109  3 分钟

给你一个长度为 n 的整数数组 nums 和 一个目标值 target。请你从 nums 中选出三个整数，使它们的和与 target 最接近。

返回这三个数的和。

假定每组输入只存在恰好一个解。

示例 1：

输入：nums = [-1,2,1,-4], target = 1
输出：2
解释：与 target 最接近的和是 2 (-1 + 2 + 1 = 2) 。

示例 2：

输入：nums = [0,0,0], target = 1
输出：0

提示：

3 <= nums.length <= 1000
-1000 <= nums[i] <= 1000
-10^4 <= target <= 10^4

我的思路：递归求所有可能组合

使用递归进行组合，每次挑选 3 个数，求这 3 个数的和，保存最小的与 target 的差值与当前的和。最后返回当前和。

class Solution {
public:
    int ret = 0;
    int minDv = INT_MAX;
    int curSum = 0;
    void func(vector<int>& nums, vector<bool>& visited, int target, int n) {
        if (n == 3) {
            if (abs(curSum - target) < minDv) {
                minDv = abs(curSum - target);
                ret = curSum;
            }
            return;
        }
        for (int i = 0; i != visited.size(); ++i) {
            if (!visited[i]) {
                visited[i] = true;
                curSum += nums[i];
                func(nums, visited, target, n + 1);
                visited[i] = false;
                curSum -= nums[i];
            }
        }
    }
    int threeSumClosest(vector<int>& nums, int target) {
        vector<bool> visited(nums.size(), false);
        func(nums, visited, target, 0);
        return ret;
    }
};

官方解法：排序 + 双指针

题目要求找到与目标值 target 最接近的三元组，这里的「最接近」即为差值的绝对值最小。我们可以考虑直接使用三重循环枚举三元组，找出与目标值最接近的作为答案，时间复杂度为 O(N^3)。然而本题的 N 最大为 1000，会超出时间限制。

那么如何进行优化呢？我们首先考虑枚举第一个元素 a，对于剩下的两个元素 b 和 c，我们希望它们的和最接近 target − a。对于 b 和 c，如果它们在原数组中枚举的范围（既包括下标的范围，也包括元素值的范围）没有任何规律可言，那么我们还是只能使用两重循环来枚举所有的可能情况。因此，我们可以考虑对整个数组进行升序排序，这样一来：

假设数组的长度为 n，我们先枚举 a，它在数组中的位置为 i；

为了防止重复枚举，我们在位置 [i + 1, n) 的范围内枚举 b 和 c。

当我们知道了 b 和 c 可以枚举的下标范围，并且知道这一范围对应的数组元素是有序（升序）的，那么我们是否可以对枚举的过程进行优化呢？

答案是可以的。借助双指针，我们就可以对枚举的过程进行优化。我们用 pb 和 pc 分别表示指向 b 和 c 的指针，初始时，pb 指向位置 i + 1，即左边界；pc 指向位置 n − 1，即右边界。在每一步枚举的过程中，我们用 a + b + c 来更新答案，并且：

如果 a + b + c ≥ target，那么就将 pc 向左移动一个位置；

如果 a + b + c < target，那么就将 pb 向右移动一个位置。

本题也有一些可以减少运行时间（但不会减少时间复杂度）的小优化。当我们枚举到 a + b + c = target 时，可以直接返回 target 作为答案，因为不会有再比这个更接近的值了。

class Solution {
public:
    int threeSumClosest(vector<int>& nums, int target) {
        sort(nums.begin(), nums.end());
        int minDv = INT_MAX;
        int curSum = 0, ret = 0;
        for (int i = 0; i != nums.size(); ++i) {
            for (int j = i + 1, k = nums.size() - 1; j < k;) {
                curSum = nums[i] + nums[j] + nums[k];
                if (abs(target - curSum) < minDv) {
                    minDv = abs(target - curSum);
                    ret = curSum;
                }
                if (curSum == target) return target;
                else if (curSum > target) {
                    --k;
                } else {
                    ++j;
                }

            }
        }
        return ret;
    }
};
